权限管理：
认证：用户的识别功能，用于识别用户和谁和可以实现哪些功能：通常的认证就是让系统知道你是谁；
授权：授权系统可以知道以后可以做什么，-让系统知道你能做什么
权限控制的方式：
    1.url的拦截：底层基于拦截器或者过滤去来实现
    通过拦截器或者过滤器拦击用户发送的请求：
    判断用户是否有权对当前的功能进行操作，
    如果有，则放行；否者就对此action进行拦截，并提示没有权限的的信息访问提示；
    2.方法注解的拦截方式的方式：
    底层是基于代理技术来实现，对action进行代理，由代理对象进行权限的控制
    public class StaffAction(){
    @HasPrivelige("delete")
    public  String deleteSatff(){

   }
    }
    //对action中的方法使用注解，起到一个标识的作用，我们的框架创建代理对象去寻找注解，由代理对象去对权限进行管理，
    如果权限足够的话则权限校验通过，如果不通过的话会抛出异常指出权限不足；
